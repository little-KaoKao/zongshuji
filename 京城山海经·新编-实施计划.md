# 《京城山海经·新编》实施计划

> **执行说明**：每一阶段按顺序完成，每一步完成后在 `- [ ]` 里打勾；每完成一个交付物立刻归档到 `/assets` 对应子目录，避免丢失中间成果。

**目标**：在 9 周内完成《京城山海经·新编》全部投稿物料——1 张总览图 + 6 张神兽图鉴 + 1 支 90 秒主视频，并完成投稿。

**方法论**：
- **风格先锁死，再批量生产**——先花一周把"青绿山水 × 数据可视化 × 工笔神兽"的风格参考图做到极致，之后所有生成以它为 style reference，保证统一。
- **单只神兽走完整管线再做下一只**——避免最后发现风格漂移要返工。每只神兽：生成 → 精修 → 高清放大 → 存档 → 再开下一只。
- **视频在图像全部定稿后才启动**——视频的神兽动态是基于已定稿图像做的"图生视频"，顺序不能颠倒。
- **频繁存档**：每完成一个版本立刻保存到 `/assets/YYYY-MM-DD_阶段_版本名/`，不覆盖旧版本。

**工具栈**：
- **AIGC 生成（主力）**：统一走 [RunningHub](https://www.runninghub.cn) 一个平台，可调用 Seedream v4/v4.5、可灵 2.5/2.6/3.0、万相 2.6、海螺 02/2.3、悠船、Veo 等所有主流模型 —— 通过本仓库的 `rh` CLI 驱动，参见 [README.md](README.md)。
- **风格一致性**：优先用 Seedream 的图生图 + 参考图（一次上传、多次引用），配合 MASTER-STYLE-REFERENCE。如需更精细控制再补 ComfyUI + IP-Adapter。
- **高清放大**：Topaz Gigapixel AI 或 Magnific（本地 / 订阅）
- **图生视频**：RunningHub 内的"可灵 2.6 pro / 万相 2.6 / 海螺 2.3 pro / Veo 3.1"按场景选用
- **平面排版**：Adobe Photoshop / Illustrator
- **视频后期**：Adobe Premiere Pro + After Effects
- **调色**：DaVinci Resolve
- **音频**：RunningHub 的 MiniMax Music 2.5 生成底乐 + Speech 2.6/2.8 HD 配旁白；后期用 Audition 混音

**迭代闭环**：`rh generate → 人工过目 → rh feedback → (AI 改 prompt) → rh regen`，每轮自动记录到 `runs.db` + `assets/<slot>/prompt-log.md`。
